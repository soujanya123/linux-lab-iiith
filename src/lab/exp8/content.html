8. Searching
Introduction

Searching is an essential functionality in any Operating System. Linux provides powerful utilities which not only searches for files and folders but also makes it easy to search for some text snippet within the content of the files in any directory. In this experiment, grep and find utilities have been explained.

Objective

The objective of this experiment is to familiarize the users with the search utilities grep and find to search string patterns and files in the machine using command line functions.

After finishing the Experiment, the user would be able to use the above-specified commands easily.

Theory

grep Command

grep is an extremely powerful search utility. grep searches the specified file for the lines matching the input pattern given as an argument to the command. By default, for each line that matches or contains a match to the input pattern, it prints that line as an output.

The syntax for using grep command is:
grep [OPTIONS] PATTERN [FILE...]
where OPTIONS are the various options available with the grep command, PATTERN is the input pattern to search for, FILE is the file containing the text which is matched against the pattern.

The input pattern should always be enclosed within single-quotes. Input pattern can also be mentioned as a regular expression.

Regular Expressions
Regular expressions are used to specify a sequence of characters given as input. To explain the usage of grep command with regular expressions, a test file example.txt has been created as shown below. 

$ cat example.txt
123456
abcdefgh
aabbaabb
abababab
ABCDefgh1234
ab*23

The rules for constructing regular expressions have been explained below.

1. Any character is a pattern that matches itself. Thus, each of the character is a regular expression in itself. 

$ grep 'a' example.txt
abcdefgh
aabbaabb
abababab
ab*23

The command matches 'a' in every line of the file. Please note that when the command is executed in the shell, the pattern given in the command is highlighted in the output. Thus, in the above example, every occurrence of 'a' is highlighted in red color in the shell.

This rule doesn't apply to some special characters like '*','.' etc. To match these special characters, backslash ('\') should precede them to escape the characters.

2. Concatenation: Concatenation is achieved by writing regular expression in sequence. For example, regular expressions 'a' and 'b' can be concatenated to form a single regular expression 'ab'.

White space characters like space are also treated as regular expressions. The strings 'ab' and 'a b' are two different regular expressions. 

$ grep 'ab' example.txt 
abcdefgh
aabbaabb
abababab
ab*23

$ grep 'a b' example.txt
$

All the lines with the pattern 'ab' are printed. But, for the second command, there is no output because there is not pattern 'a b' in the file.

3. Brackets ([]): A list of characters enclosed within '[' and ']' matches any single character in that list. 

$ grep '[abcd]' example.txt
abcdefgh
aabbaabb
abababab
ab*23

4. The meta character '-' is used to specify the range of characters for pattern matching. For example, if a string might include any alphabet, regular expression to match the pattern is written as [a-z]. 

$ grep '[0-9]' example.txt
123456
ABCDefgh1234
ab*23

The first command matches the lines with numeric digits in it.

5. The meta characters '^' and '$' are used to match the beginning and end strings or patterns.
^ - matches the characters or string at the beginning of the line
$ - matches the characters or string at the end of the line 

$ grep '^ab' example.txt
abcdefgh
abababab
ab*23

$ grep 'ab$' example.txt
abababab

The first command in the above example outputs the lines which start with 'ab'. The second command outputs the lines which end with 'ab' string.

6. Dot Character: The '.' meta character is used to match any single character. For example, the following command matches any pattern which start with 'a' and ends with '3' and has any three characters in between. 

$ grep 'a...3' example.txt
ab*23

7. Repititions: The following operators repeat the occurrence of any character to which they are applied.
* - matches preceding pattern zero or more times
? - matches preceding pattern at most one time
+ - matches preceding pattern at least once
{n} - matches preceding pattern exactly n times
{n,} - matches preceding pattern n or more times
{n,m} - matches preceding pattern at least n and at most m times

Note: Except '*', all the operators have to be preceded with backslash ('\'). 

$ grep 'a*' example.txt
123456
abcdefgh
aabbaabb
abababab
ABCDefgh1234
ab*23

The above command outputs all the strings which start with a zero or more times. 

$ grep '^ab*' example.txt
abcdefgh
aabbaabb
abababab
ab*23

The above command outputs all the strings which start with 'a' and is followed by 'b' zero or more times. 

$ grep '^\(ab\)\(ab\)*' example.txt
abcdefgh
abababab
ab*23

The above command searches for all the patterns in the text file which start with 'ab' and may or not have any more 'ab' pattern. 

$ grep '^ab\?' example.txt
abcdefgh
aabbaabb
abababab
ab*23

The above command searches for all the patterns which start with 'a' and followed by zero or one 'b'. 

$ grep '^ab\+' example.txt
abcdefgh
abababab
ab*23

The above command searches for all the patterns which start with 'a' and followed by at least one 'b'. 

$ grep '.\{10\}' example.txt
ABCDefgh1234

The above command searches for a string exactly 10 characters long. 

$ grep '.\{6,\}' example.txt
123456
abcdefgh
aabbaabb
abababab
ABCDefgh1234

The above command searches for a string at least 6 characters long. 

$ grep '.\{6,10\}' example.txt
123456
abcdefgh
aabbaabb
abababab
ABCDefgh1234

The above command searches for a string at least 6 characters long and at most 10 characters long.

8. Parentheses '()' can be used to denote a regular expressions. 'abcd' is concatenation of four regular expressions 'a', 'b', 'c'and 'd', but '(ab)cd' is a concatenation of three regular expressions 'ab', 'c' and 'd'.

Note: Parentheses are treated as part of the string unless they are escaped with backslash('\'). 

$ grep '\(ab\)cd' example.txt
$ grep '\(ab\)*cd' example.txt
abcdef gefrd geef
(ab)cd
abghefcd
aaaaaaacd
bbbbbbbcdfeg
ababababcdefgh
abcdefgh
abcdefgh

Since, there is no 'ab' string enclosed within brackets, the first command does not give any output. But, the second command matches the string pattern given because of the use of backslash before the parentheses.

Options

Grep can be used with the following options:
v - it reverses the pattern specified and prints all the lines that DO NOT match the specified regular expression 

$ grep '\(1234\)' example.txt
123456
ABCDefgh1234

$ grep -v '\(1234\)' example.txt
abcdefgh
aabbaabb
abababab
ab*23

Compare the above commands and note the difference.

A (num) - grep takes a numeric value num as an argument and prints num lines of the file text after the pattern match 

$ grep -A 2 'ab*' example.txt
abcdefgh
aabbaabb
abababab
ABCDefgh1234
ab*23

In the above output, the command prints two lines after every pattern match.

B (num) - as opposed to the A option, this option prints num lines of text in the file BEFORE the pattern match 

$ grep -B 2 'ab*' example.txt
123456
abcdefgh
aabbaabb
abababab
ABCDefgh1234
ab*23

c - it prints the number of lines matching the pattern specified 

$ grep -c 'ab*' example.txt
4

o - the option prints only the matched patterns 

$ grep -o 'ab*' example.txt
ab
a
abb
a
abb
ab
ab
ab
ab
ab

find Command

The find command is used to locate files on Linux system. The command searches for the set of directory specified in the argument for the files. The syntax for find is:
find [path] [option] [file name / expression]
where path is the directory where the file is to be located, file name is the file to be located and expression are the regular expressions which can be used for pattern matching.

Executing find in the terminal lists down all the files in the current directory. The command recursively searches all the files and folders in the directory. 

$ ls
abcd.org test/

$ find
.
./test
./test/dir
./test/dir/abc.org
./test/image2.jpg
./test/image1.jpg
./test/abc.tar
./test/abc.zip
./test/abd
./test/abc.org
./abcd.org

find command provides many options for use. In this tutorial, only two commonly used options would be explained. For further information, please refer to the man page of find.
iname - takes an argument and recursively searches the specified directory for the given file name or pattern
delete - sometimes, the files no longer needed have to be deleted. So, find command is used to locate them and delete is used to delete the files 

$ find -iname abc.org
./test/dir/abc.org
./test/abc.org

To find the files in a specified directory, 

$ find test/ -iname abc.org
test/test/dir/abc.org
test/test/abc.org

To search for a file using a regular expression pattern, 

$ find test/ -iname 'ab*'
test/test/dir/abc.org
test/test/abc.tar
test/test/abc.zip
test/test/abd
test/test/abc.org
test/abcd.org

To delete the specified file, 

$ find test/ -iname abc.org -delete

Please note that the delete Option should be used with caution. It should always be at the end of the command arguments. If the delete option is used before iname, all the files and folders would be deleted. The command does not ask for permission or provide any warning, whatsoever, before execution.

The grep and find commands have been explained in the theory section. Now, the user should be able use these commands to locate files in the directories easily. For further information on the commands, please refer to the Further Readings Section.
